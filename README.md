### 🧪 Manual Testing Expertise

<div align="center">

| 🔬 Testing Type | 📋 Description |
|:---|:---|
| ✅ **Functional Testing** | Verifying every feature works as per requirements |
| 🔁 **Regression Testing** | Ensuring new changes don't break existing features |
| 💨 **Smoke Testing** | Quick sanity check after every new build |
| 🧹 **Sanity Testing** | Focused testing on a specific bug fix or feature |
| 🔍 **Exploratory Testing** | Unscripted testing to discover unexpected bugs |
| 👤 **UAT (User Acceptance)** | Validating software from end-user perspective |
| ⚡ **Performance Testing** | Testing speed, load, and stability under stress |
| 🔒 **Security Testing** | Identifying vulnerabilities and data exposure risks |
| 📱 **Compatibility Testing** | Cross browser, cross device, cross OS validation |
| 🔗 **Integration Testing** | Testing interaction between modules and services |
| 🔄 **Re-testing** | Verifying reported bugs are fixed correctly |
| 📦 **Build Acceptance Testing** | Go/no-go validation before test cycle begins |
| 📊 **Boundary Value Analysis** | Testing at edge values of input ranges |
| ⚖️ **Equivalence Partitioning** | Grouping inputs into valid and invalid partitions |
| 🌐 **API Testing** | Validating REST APIs using Postman |
| 📝 **Documentation Testing** | Reviewing test plans, cases, and bug reports |
